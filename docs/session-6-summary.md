# Итоги шестой сессии — Zerocoder полностью + bulk import + save-to-dict

## Контекст

В этой сессии добавили Zerocoder со всеми 7 типами задач из его регламента, доработали мастер-импорт сайтов в админке, и реализовали сохранение метки в справочник прямо из мастера. Локально всё работает, на проде застряло — Vercel-алиас не обновляется (см.раздел про деплой ниже).

## Что сделано

### Фаза A: Сохранение метки в справочник из мастера

[`CustomSiteFinishButtons.tsx`](../src/components/CustomSiteFinishButtons.tsx) + [`wizard-actions.ts`](../src/lib/wizard-actions.ts):
- В «Свой вариант» шага сайта вместо одной кнопки «Дальше →» теперь две: **💾 Сохранить в справочник** и **➡️ Использовать один раз**
- «Сохранить» записывает URL+тег в `sites` текущей компании с audit-логом (откатываемо)
- «Использовать один раз» — обычный путь без записи в БД
- Работает во всех 4 ветках мастера (+ позже добавилось в новых для Zerocoder)
- Без admin-auth: внутренний инструмент, для команды

### Фаза A: Массовый импорт сайтов в админке

[`/admin/sites/BulkImport.tsx`](../src/app/admin/sites/BulkImport.tsx) — отдельная кнопка «📋 Импорт списком» рядом с «+ Добавить»:
- Вставка из буфера: CSV или TSV (для копирования из Google Sheets)
- Автоопределение разделителя
- Превью с пометкой статусов по строкам: новый / совпадает / конфликт / некорректный
- При конфликтах — выбор «пропустить / перезаписать тег»
- Все сохранения в audit_log

### Фаза B: Zerocoder базовые ветки + плейсхолдеры

- **Кодирование плейсхолдеров** в URL: `{campaign_id}`, `{ad_id}`, `{keyword}` (для Я.Директ/FB) теперь остаются литералом в URL, не кодируются как `%7B...%7D`. Реализовано в [`lib/utm.ts`](../src/lib/utm.ts) и [`EditableUtm.tsx`](../src/components/EditableUtm.tsx) через `encodePreservePlaceholders`.
- **Типы задач data-driven**: вместо хардкода — список с фильтром по компании. У Matrius все 5 типов, у Zerocoder — 8 типов из его регламента (без СММ).
- **Засев Zerocoder данных**: 47 каналов всего:
  - Анонсы — 12 (Почта, ТГ-боты × 7, Max, Пуш, Инстаграм, YouTube)
  - Гайды — 3 (Почта, ТГ-бот, ВК-бот)
  - Реклама — 14 (по людям: Вика, Vlad, Valeriy, Дмитрий, Александр, Владимир, Филипп, Роман)
- **Гайды для Zerocoder** — `utm_content=guide` фикс. Реализовано через autoskip placement (если в БД 1 место — пропускается шаг)

### Фаза C: Сложные ветки Zerocoder

[`BranchSimple.tsx`](../src/components/BranchSimple.tsx) — универсальная ветка «канал → сайт → результат» (без даты), используется для:
- **Воронка вебинара** — 10 каналов с группировкой по «Шаг/письмо» (Продажное / Вход на вебинар / Шальной), каналы пред-заполнены `utm_content` и `utm_term`
- **Внешние выступления** — 2 канала (Спикерство, Подкаст), `utm_content=спикер` (юзер заменяет на имя в паровозике)
- **Блог** — 1 канал (Журнал), `utm_source=magazine`
- **В Геткурсе** — 5 каналов (тренинги, вебинар, интенсив, урок, баннер)

**Новые колонки в схеме `channels`**: `default_content`, `default_term` — заполняются при сидинге, используются в результате мастера. Миграция применяет ALTER TABLE ADD COLUMN с обработкой «уже существует».

## Что нашли

### Vercel deploy stuck

Все деплои сегодня застряли в статусе `UNKNOWN` — судя по всему, нестабильная сеть рвёт upload. Промот в production алиас (`utm-matrius.vercel.app`) не выполняется. Алиас остался указывать на самый первый деплой 9-часовой давности — там нет ни bulk import, ни save-to-dict, ни Zerocoder каналов.

В качестве workaround сделан empty commit + push в GitHub — это триггерит auto-deploy с Vercel-стороны (без upload с твоей машины). Этот деплой собирается прямо сейчас.

### Кириллица в Cyrillic-named channels

`group_name` каналов Zerocoder ads хранит имена людей кириллицей: «Вика», «Дмитрий» и т.д. Работает, в URL не попадает (только в UI).

### YouTube канал Zerocoder

В регламенте у YouTube `utm_medium=пусто`, `campaign=пусто`, content/term тоже пусты. Реализован как канал с `needs_manual_medium=1` — пользователь сам вводит medium при выборе.

## Как проверить локально (пока ждём прод)

**Dev-сервер у тебя должен быть запущен**: открой http://localhost:3000 — там вся последняя версия кода + база Turso (продовая) с засеянными данными для обеих компаний.

### Что протестировать

1. **Matrius → Анонс → Email → Unisender → выбрать сайт → дата** → готовая UTM
2. **Matrius → СММ → ВК-пост → сайт → дата** → проверить `utm_content=post`
3. **Matrius → Гайд → ввести «Скорочтение» → выбрать вариант slug → сайт → место → дата** → проверить `utm_source=pdf`
4. **Matrius → Реклама → Артём → Я.Директ → сайт** → проверить что `utm_campaign` = хвост URL (не тег)
5. **Zerocoder → Анонс** → теперь должны быть варианты (12 каналов вместо «правила не настроены»)
6. **Zerocoder → Воронка вебинара → Продажное письмо → ТГ-бот → сайт** → в результате должно быть `utm_content=funnel`, `utm_term=message-N`
7. **Zerocoder → Реклама → Вика → Я.Директ → сайт** → должны быть Yandex-плейсхолдеры (в паровозике замени значения)
8. **Свой вариант сайта** в любой ветке → ввести URL → увидеть 2 кнопки (Сохранить / Один раз)
9. **/admin/sites** (логин `matrius2026`) → должна быть кнопка «📋 Импорт списком» → вставить CSV/TSV → превью → импорт
10. **Переключатель компании** в шапке админки — переключить на Zerocoder, увидеть его 47 каналов
