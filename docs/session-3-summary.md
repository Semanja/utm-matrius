# Итоги третьей сессии — мастер целиком + темы

## Контекст

Продолжение из второй сессии. Цель — собрать все 4 ветки мастера (Анонс, СММ, Гайд, Реклама), реализовать ручное редактирование готовой UTM, поддержку «свой вариант» на каждом шаге, и подключить переключатель тем.

## Что сделано

### Мастер UTM

**Базовая навигация** ([Wizard.tsx](../src/components/Wizard.tsx)):
- Шаг 0: выбор компании (Matrius / Zerocoder)
- Шаг 1: выбор типа задачи
- Если компания не настроена (Zerocoder) — fallback на ручную форму
- «Свой вариант» в типе задачи → ручная форма

**Ветка Анонс** ([BranchAnonse.tsx](../src/components/BranchAnonse.tsx)):
- Шаг 2 платформа (Email / Telegram-бот / VK-бот / Max-бот / свой)
- Шаг 3 канал (подвыбор; если у платформы 1 канал — пропускается автоматом)
- Шаг 4 сайт (38 штук с поиском или свой URL)
- Шаг 5 дата (календарь сегодня → +14 дней)
- Результат: `utm_campaign` = хвост URL, `utm_content = announce`, `utm_term = date_дд.мм.гг`

**Ветка СММ** ([BranchSmm.tsx](../src/components/BranchSmm.tsx)):
- Шаг 2 канал (ВК-пост / ТГ-пост / Max-пост / свой)
- Шаг 3 сайт, шаг 4 дата — как в Анонсе
- Результат: `utm_content = post`, `utm_term = date_дд.мм.гг`, `utm_campaign` = хвост URL

**Ветка Гайд** ([BranchGuide.tsx](../src/components/BranchGuide.tsx)):
- Шаг 2 название документа: ввод на русском → 4 варианта транслитерации (дефисы, без разделителей, подчёркивания, кириллица) + свой вариант → `utm_medium`
- Шаг 3 сайт → `utm_campaign` = тег из справочника (не хвост URL!)
- Шаг 4 место размещения (link-text / qr / image / button / свой) → `utm_content`
- Шаг 5 дата (месяц) → `utm_term = мм.гг`
- `utm_source = pdf` (фикс)

**Ветка Реклама** ([BranchAds.tsx](../src/components/BranchAds.tsx)):
- Шаг 2 кто настраивает (6 человек + «Другое»)
- Шаг 3 канал: если 1 — автомат, иначе выбор; для «Другое» — список 10 внешних сервисов с ручным вводом medium
- Шаг 4 сайт; для каналов с хвостиком (Илья, Евгения-VK-ads, Ирина) — medium = `prefix_` + хвост
- Спец-случай: для Артёма-Я.Директа `utm_campaign` = хвост URL, для всех остальных — тег из справочника
- Поля `utm_content` и `utm_term` остаются пустыми — заполнить через «✏️ Редактировать»

### Готовая UTM с inline-редактированием

[EditableUtm.tsx](../src/components/EditableUtm.tsx):
- Финальная панель с готовой ссылкой и кнопкой «📋 Скопировать»
- Кнопка «✏️ Редактировать вручную» → ссылка становится «паровозиком»: статичные части (`?utm_source=`, `&utm_medium=` …) + поля ввода между ними
- Ширина инпутов подстраивается под содержимое
- «↺ Вернуть как было» — откат к значениям из мастера

### Темы оформления

[globals.css](../src/app/globals.css) + [ThemeSwitcher.tsx](../src/components/ThemeSwitcher.tsx):
- 4 палитры: Светлая (фиолетовый акцент), Тёмная (чёрно-жёлтая), Zerocoder (тёмная с мятным + фиолетовыми заголовками), Matrius (синий фон + оранжевый акцент)
- CSS-переменные: `--bg`, `--surface`, `--text`, `--text-muted`, `--border`, `--accent`, `--accent-hover`, `--accent-text`
- Дополнительно: в Zerocoder и Matrius заголовки H1/H2 имеют отдельные брендовые цвета (фиолетовый и синий) через `[data-theme] h1` правила
- Базовый размер шрифта увеличен: `html { font-size: 19.25px }` (на ~20% больше браузерного дефолта 16px) для лучшей читаемости на цветных темах
- Переключатель в шапке (4 кнопки)
- Inline-скрипт в [layout.tsx](../src/app/layout.tsx) ставит `data-theme` до рендера body — без вспышки
- Выбор сохраняется в localStorage (`utm-gen-theme`)
- Статусные цвета (зелёный для готовой UTM, жёлтый для предупреждений, красный для ошибок) — полупрозрачные, читаются на любой теме

## Что нашли

### Логика utm_term — пробел vs подчёркивание

В исходном ТЗ значение было `date nn.nn.nn` с пробелом. Пробел в URL кодируется как `%20`. По согласованию заменили на подчёркивание: `date_дд.мм.гг`. URL чище, в аналитике значение читается так же.

### utm_campaign — две разные стратегии

- **Анонс, СММ**: `campaign` = хвост URL (path после слеша, или домен если пусто). Тег из справочника здесь только для визуальной навигации.
- **Гайд, Реклама**: `campaign` = тег из справочника.
- **Исключение в Рекламе**: для Артёма-Я.Директа `campaign` снова = хвост URL. Зашито хардкодом в коде (`isYandexArtem`), не в БД. Если такая логика появится у других — стоит вынести в отдельную колонку channels.

### VK-бот — на самом деле один

В исходной таблице было 2 VK-бота (matrius_ru и gc_matrius_ru). Пользователь подтвердил, что нужен только `matrius_ru`. Убрали из сидинга. Заодно реализовали логику автопропуска шага: **если у платформы только 1 канал, выбор каналов не показывается** — канал ставится автоматом, юзер идёт сразу к сайту.

### Транслитерация vs семантический перевод

Пользователь изначально хотел «понятные сокращения» (скорочтение → speedreading), а не транслит. Без LLM такое сделать корректно нельзя. Договорились оставить транслитерацию + свой вариант на этом шаге Гайда.

### EditableUtm — собирает URL вручную

Решение использовать `encodeURIComponent` для значений + ручной конкатенации, а не `URL.searchParams`. Так пользователь может вводить любые символы в полях «паровозика», даже невалидные — отображаются как есть.
