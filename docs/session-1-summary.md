# Итоги первой сессии — планирование UTM-генератора

## Контекст

Стартовала разработка веб-приложения для генерации UTM-меток с админкой правил и пошаговым клиентским мастером. Холдинг = две компании: Matrius (взрослое направление? нет — детское) + Zerocoder (взрослое). Сначала запускаем для Matrius, Zerocoder — заглушкой.

## Что сделано

- Согласован весь продуктовый план (см. [utm-generator-plan.md](utm-generator-plan.md))
- Финализировано ТЗ мастера со всеми ветками: Анонс, Гайд, Реклама, СММ (см. [wizard-spec.md](wizard-spec.md))
- Финализировано ТЗ админки: разделы, операции, журнал изменений + откат (см. [admin-spec.md](admin-spec.md))
- Получен и сохранён справочник «Сайты → теги» Matrius (`data/site-tags.json`)
- Согласованы 4 цветовые темы (светлая, тёмная, Zerocoder, Matrius) с HEX-кодами
- Установлены CLI: `gh` (GitHub), `vercel` (Vercel)
- Создана продовая БД на Turso через веб-дашборд (URL и токен у пользователя)
- Стек подтверждён: Next.js + Turso + Vercel, авторизация админки логин/пароль

## Что нашли

### Турсо CLI не ставится стандартным способом

Официальный PowerShell-инсталлер по `get.tur.so/install.ps1` отдаёт 404. Обошли — БД создаётся через веб-дашборд за минуту, URL+токен копируется в `.env.local`. CLI был для удобства, не блокер.

### create-next-app конфликтует с существующими папками

`npx create-next-app .` отказывается работать, если в папке уже есть `assets/`, `data/` (даже если они не конфликтуют по именам). Нужно либо ставить в подпапку, либо временно убирать наши папки.

### Поведенческое: не уходить в действия молча

В моменте, когда тулы выдают ошибки, нужно сразу останавливаться и объяснять. Уход в «починю молча» создаёт ощущение пропавшей коммуникации. Зафиксировано в памяти.

## История итераций

| Что попробовали | Результат |
|---|---|

| Одна компания в системе | Заменили на две: Matrius + Zerocoder (заглушка) |
