# Веб-приложение для генерации UTM-меток с админкой правил

## Контекст

Нужно веб-приложение, которое генерирует UTM-метки по корпоративным правилам разметки. Проблема: правила есть, но команда их путает, забывает или не знает. Цель — пошаговый мастер, который ведёт сотрудника по правилам и подставляет значения автоматически, плюс админка для самих правил, чтобы их можно было править без программиста.

## Что сделано

- Согласован общий замысел приложения
- Выбран стек: **Next.js + Turso** (облачный SQLite), хостинг **Vercel**, админка под **логином и паролем**
- Согласованы 4 цветовые темы (светлая, тёмная, Zerocoder, Matrius)
- Финализировано ТЗ мастера со всеми ветками и значениями — см. [wizard-spec.md](wizard-spec.md)
- Получен и сохранён справочник «Сайты → теги» Matrius — см. `data/site-tags.json`

## Что нашли

### Технический нюанс по БД

SQLite в чистом виде на Vercel не работает — Vercel serverless, файловая система между запросами не сохраняется. Используем **Turso** — облачный SQLite с бесплатным тарифом, синтаксис тот же.

### Логика мастера (финализирована)

Полное ТЗ мастера со всеми шагами, ветвлениями и значениями — отдельный документ: [wizard-spec.md](wizard-spec.md).

Кратко:
- **Шаг 0:** выбор компании (Matrius / Zerocoder)
- **Шаг 1:** тип задачи (Анонс / Гайд / Реклама / СММ / свой вариант)
- Дальше — своя ветка для каждого типа задачи
- На каждом шаге — опция «свой вариант»
- Сайт выбирается из справочника, `utm_campaign` подставляется автоматом
- Для нейро-страниц `utm_campaign = neyro` (направление выбирается на самом лендинге, не в мастере)

### Две компании: Matrius и Zerocoder

Холдинг состоит из двух направлений — взрослое (Zerocoder) и детское (Matrius). У каждой компании свои правила разметки UTM. На старте мастера пользователь выбирает компанию, дальше шаги отрисовываются по правилам выбранной компании.

- **Matrius** — правила есть, заполняем сейчас
- **Zerocoder** — правила появятся позже, пока заглушка

### Справочник «Сайты → теги»

Отдельная таблица в админке: соответствие лендинга/страницы и его тегов (`utm_campaign` и, возможно, другие). При выборе сайта в мастере подставляются автоматически.
- Заполняется и правится в админке
- Используется в шагах мастера, где раньше пользователь вводил «хвост ленда» / «тег активности» вручную
- Структура колонок — финализирую, когда придёт таблица

### Поведение пустой админки (fallback)

Если у выбранной компании в админке нет ни одного правила:
- Показать заголовок «Правила для [компания] ещё не настроены»
- Подсказка: «Настройте справочники в админке или сгенерируйте метку вручную ниже»
- Под подсказкой — форма со всеми стандартными UTM-полями (source, medium, campaign, content, term) как обычные текстовые поля
- Кнопка «Собрать UTM» формирует строку из введённого вручную

## Архитектура (краткая схема)

- **Frontend (Next.js)** — клиентский мастер + админка в одном приложении
- **Backend (Next.js API routes)** — CRUD справочников, валидация, выдача правил мастеру
- **БД (Turso)** — справочники, правила, опционально логи генераций
- **Auth (NextAuth)** — пароль для админки
- **Hosting (Vercel)** — деплой из GitHub, домен `*.vercel.app` (можно подключить свой)
