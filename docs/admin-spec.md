# ТЗ админки UTM-генератора

## Уровень гибкости

**Базовый**: админ редактирует содержимое справочников. Структура веток мастера (Анонс / Гайд / Реклама / СММ) и спец.логика — в коде.

## Авторизация

Один логин/пароль на всю админку. Сессия сохраняется до явного выхода.

## Разделы

### 1. Сайты и теги
- Таблица: URL, тег (utm_campaign), компания
- Действия: добавить, править, удалить, импорт CSV
- Поиск по URL и тегу

### 2. Каналы — Анонсы
- Таблица: платформа, название канала, source, medium, компания
- Группировка по платформе (Email / Telegram-бот / VK-бот / Max-бот)

### 3. Каналы — СММ
- Таблица: канал, source, medium, компания

### 4. Каналы — Реклама (персональные)
- Таблица: человек, название канала, source, medium, нужен ли хвостик сайта в medium
- Действия: добавить, править, удалить

### 5. Каналы — Реклама (общие, «Другое»)
- Таблица: название, source, ввод medium вручную (да/нет)

### 6. Люди (рекламщики)
- Список людей и привязки к каналам
- Добавление нового человека = выбор каналов, с которыми он работает

### 7. Места размещения (гайд)
- Список: link-text, qr, image, button, и любые новые

### 8. Компании
- Matrius / Zerocoder
- Переключатель компании в шапке админки — все справочники меняются на справочники выбранной компании

## Общая логика операций

### Сохранение
- Применяется сразу (без черновиков)
- Старая версия записи остаётся в журнале

### Удаление
- Подтверждение «Точно удалить?»
- Soft delete (помечается как удалённое, но остаётся в журнале — можно восстановить)

### Валидация
- Source/medium не пустые
- URL валидный (для справочника сайтов)
- Тег уникален в пределах URL (одно правило на URL)
- Имена каналов уникальны в пределах ветки

## Журнал изменений и откат

### Что логируется
Каждая правка/добавление/удаление в любом справочнике:
- Дата и время
- Тип операции (создание / правка / удаление / восстановление)
- Что было до и что стало после
- (Поле «кто» — пока всегда один админ, но поле в БД есть на будущее)

### Где смотреть
- У каждой записи в таблице — кнопка «История»
- Открывается список всех версий этой записи

### Откат
- В истории — кнопка «Откатить к этой версии»
- Восстанавливает старое значение
- Сам откат тоже логируется как операция

## Импорт CSV (только «Сайты и теги»)
- Кнопка «Импорт CSV»
- Формат: колонки `url, tag`, разделитель — запятая
- Превью перед импортом
- При конфликте (URL уже есть с другим тегом) — выбор: пропустить / перезаписать

## Что НЕ редактируется через админку
- Структура веток мастера (4 типа задач и их шаги)
- Темы оформления
- Сами поля UTM (source/medium/campaign/content/term — стандарт)
- Спец.логика (хвостик сайта, транслитерация в гайде) — только в коде

Если позже что-то из этого нужно сделать редактируемым — переносим отдельной задачей.
