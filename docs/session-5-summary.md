# Итоги пятой сессии — bulk import + save-to-dictionary, Zerocoder в плане

## Контекст

Промежуточная сессия. Сделали две независимые фичи и начали готовиться к большой задаче — добавить Zerocoder с его специфической структурой меток.

## Что сделано

### Массовый импорт сайтов в админке

[`/admin/sites/BulkImport.tsx`](../src/app/admin/sites/BulkImport.tsx) + actions:
- Кнопка «📋 Импорт списком» рядом с «+ Добавить»
- Вставка из буфера: CSV (запятая) или TSV (таб из Google Sheets) — авто-определение
- Парсер: пропускает заголовок если первая строка `url\t...` или `url,...`
- Превью с пометкой статусов:
  - **новый** — добавим
  - **совпадает** — уже есть с таким же тегом, пропустим
  - **конфликт** — есть с другим тегом, дать выбор
  - **некорректный** — пустой URL/тег
- При наличии конфликтов — переключатель «пропустить / перезаписать тег»
- После импорта — отчёт с количествами + лог ошибок (если были)
- Каждое создание/обновление пишется в `audit_log` (откатываемо)

Используется на `/admin/sites` для обеих компаний (учитывает выбранную в шапке).

### «Сохранить в справочник» прямо из мастера

[`CustomSiteFinishButtons.tsx`](../src/components/CustomSiteFinishButtons.tsx) + [`wizard-actions.ts`](../src/lib/wizard-actions.ts):
- В шаге «Сайт» любой ветки (Анонс / СММ / Гайд / Реклама), когда выбран «Свой вариант» и заполнены поля — вместо одной кнопки «Дальше →» появляются две:
  - **💾 Сохранить в справочник** — записать URL+тег в `sites` текущей компании и продолжить мастер
  - **➡️ Использовать один раз** — пропустить сохранение, просто собрать UTM для этой генерации
- Для Анонс/СММ тег извлекается автоматически из URL (хвост), пользователь только URL вводит
- Для Гайд/Реклама — пользователь вводит и URL, и тег
- Server action `saveSiteFromWizard()` без требования admin-auth (мастер публичный)
- Дубликаты обрабатываются: если URL уже есть с тем же тегом — ничего не делает; если с другим — обновляет; если был удалён — восстанавливает и обновляет тег
- Все сохранения попадают в `audit_log` с пометкой `via: "wizard"`, откатываются через `/admin/history`

## Что нашли

### Структура Zerocoder сильно отличается от Matrius

Файл регламента ([`Регламент UTM-меток ...html`](C:\Users\seman\Downloads\Регламент%20UTM-меток%2096e412e324dd494db411f03c7d948f25.html)) содержит **7 типов задач** против 4 у Matrius:

| Блок | Каналов | Особенность |
|---|---|---|
| Анонсы | 12 | Похоже на Matrius |
| **Воронка вебинара** | ~10 | Доп.шаг «Шаг/письмо» (Продажное / Вход / Шальной), content разный |
| **Внешние выступления** | 2 | Спикерство/Подкаст, content=спикер (имя) |
| **Блог** | 1 | source=magazine |
| Реклама | 15 | Плейсхолдеры `{campaign_id}` `{ad_id}` `{keyword}` |
| **В Геткурсе** | 5 | source=getcourse/site, разные content |
| Гайды | 3 | Проще, чем у Matrius |

**СММ у Zerocoder нет** (это часть Анонсов).

### Тег = хвост URL у Zerocoder

В отличие от Matrius (где `show-math-flow-1` ≠ URL slug), у Zerocoder тег полностью совпадает с хвостом URL. Это значит: справочник сайтов = справочник тегов, и пользователь Zerocoder может ввести URL → тег извлечётся автоматически. Не нужно заранее сидить большой справочник.

### Архитектурная проблема: типы задач хардкод

Сейчас `Wizard.tsx` хардкодит 4 ветки (announce/smm/guide/ads/custom). У Zerocoder нужны другие. Чтобы поддержать обе компании — типы задач нужно вынести в БД и сделать data-driven. Пока не сделано.
